# Лабораторная работа №23. Интерактивные веб-приложения на JavaScript

## Описание игры

**Правила игры**: пользователю необходимо заполнить своими символами в процессе игры либо диагональ, либо вертикаль, либо горизонталь игрового поля для выигрыша

**Логика победы/поражения**

Если пользователь одним символом три клетки из числа выигрышных комбинаций, тогда пользователь признается победителем. В таком случае второй игрок признается проигравшим.

**Особенности реализации**

Для реализации приложения были использованы механизмы получения элементов DOM, настроены обработчики событий для отслеживания кликов по клеткам игрового поля, а также функции проверки победителей и перезапуска.

## Игровая механика

**Как определяется победитель?**

Победитель объявляется по перебору всех выигрышных комбинаций одним символом, если таковая обнаружена - на флаге устанавливается true, и проверка прекращается

**Как обрабатываются клики?**

Для обработки клика была создана отдельная функция `function handleCellClick(event)`: определяет, какая клетка была нажата, получает индекс этой клетки и проверяет, можно ли сделать ход. Если клетка свободна, тогда ставится X или O

**Как работает перезапуск?**

Перезапуск начинает с игрока "X", очищает игровое поле, активирует игру и удаляет все символы X и O из клеток

## Структура данных

**Массив board** - это массив из 9 пустых строк, представляющий игровое поле, где каждый элемент соответствует клетке.

**Условия победы** отражены в массиве всех возможных выигрышных комбинаций

```
const winConditions = [
    [0, 1, 2],
    [3, 4, 5],
    [6, 7, 8],
    [0, 3, 6],
    [1, 4, 7],
    [2, 5, 8],
    [0, 4, 8],
    [2, 4, 6],
];
```

**Хранение состояния игры** указывается во флаге `gameActive` (true = игра идёт, false = игра окончена)


## Алгоритмы

**Алгоритм проверки победы**: проверяем все 8 выигрышных комбинаций (все три клетки заполнены одним символом). Если находит комбинацию - устанавливает флаг победы на true и прекращает проверку

**Алгоритм определения ничьи**: если в массиве не осталось пустых клеток, тогда объявляется ничья, обновляется статус и останавливается игра

**Алгоритм смены игрока**: если никто не победил и нет ничьи - происходит переключение текущего игрока посредством сброса и очищения игрового поля и обновлением статуса
